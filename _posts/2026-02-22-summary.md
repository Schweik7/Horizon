### [Andrej Karpathy talks about "Claws"](https://simonwillison.net/2026/Feb/21/claws/#atom-everything) ⭐️ 8.0/10

Andrej Karpathy identified and endorsed "Claws" as a new architectural layer in AI systems, describing them as a layer above LLM agents that advances orchestration, scheduling, context management, tool calling, and persistence. He specifically mentioned implementations like NanoClaw, which is a ~4000-line core engine that runs agents in containers by default, and noted other examples including nanobot, zeroclaw, ironclaw, and picoclaw. This matters because it signals an important evolution in AI system architecture, moving from standalone LLM agents to more sophisticated, persistent, and orchestrated systems that can run on personal hardware. Karpathy's endorsement as a leading AI researcher lends credibility to the concept and could accelerate adoption and standardization of this new layer across the AI development community. A key technical detail is that NanoClaw's core engine is approximately 4000 lines of code, which Karpathy highlights as being manageable, auditable, and flexible. Furthermore, OpenClaw, a foundational example, uses a hub-and-spoke architecture with a central Gateway WebSocket server that routes messages from various platforms (like WhatsApp, iMessage, Slack) to an Agent Runtime.

*Source: rss/Simon Willison*

**Background**: LLM agents are AI systems built on top of large language models that can take actions, such as using tools or APIs, to complete tasks. Orchestration refers to the coordination and management of these agents' workflows, including scheduling tasks and maintaining context across interactions. The term "Claws" is emerging as a category for persistent, personal AI agent systems that integrate with messaging protocols and can both act on instructions and schedule tasks autonomously.

### [Taalas serves Llama 3.1 8B at 17,000 tokens/second](https://simonwillison.net/2026/Feb/20/taalas/#atom-everything) ⭐️ 8.0/10

Canadian hardware startup Taalas announced its first product, a custom hardware implementation of Meta's Llama 3.1 8B model that can run inference at 17,000 tokens per second. This performance is achieved through an 'aggressively quantized' design combining 3-bit and 6-bit parameters. This represents a potential paradigm shift in efficient AI deployment, as such extreme speeds could enable real-time, high-volume generative AI applications previously limited by inference latency. It demonstrates the significant performance gains possible through custom silicon designed specifically for quantized models, which could lower the cost and energy footprint of running LLMs at scale. The implementation uses an aggressive quantization scheme that mixes 3-bit and 6-bit parameters, a technique that reduces model size and computational requirements but can impact accuracy. Taalas has indicated their next-generation hardware will use 4-bit quantization, suggesting they have a long hardware development cycle and are iterating on the precision-performance trade-off.

*Source: rss/Simon Willison*

**Background**: Quantization is a technique that reduces the numerical precision of a model's parameters (e.g., from 32-bit floating point to 3-bit integers) to shrink its memory footprint and accelerate computation, often with a trade-off in accuracy. Llama 3.1 8B is a popular, open-source large language model with 8 billion parameters, released by Meta in July 2024. Custom hardware like Application-Specific Integrated Circuits (ASICs) or Neural Processing Units (NPUs) is designed to execute specific AI workloads, like LLM inference, far more efficiently than general-purpose CPUs or GPUs.

### [ggml.ai joins Hugging Face to ensure the long-term progress of Local AI](https://simonwillison.net/2026/Feb/20/ggmlai-joins-hugging-face/#atom-everything) ⭐️ 8.0/10

Georgi Gerganov's ggml.ai organization, which created the foundational llama.cpp project, has officially joined Hugging Face as of February 2026, with the stated goal of ensuring the long-term progress of local AI infrastructure. This move consolidates two of the most influential open-source projects in AI—llama.cpp and Hugging Face's Transformers library—under one stewardship, which could significantly accelerate the development and accessibility of local AI by aligning the dominant model definition framework with the leading local inference engine. The joint roadmap includes creating seamless 'single-click' integration between the Transformers library and the ggml ecosystem and improving the packaging and user experience of ggml-based software to make local inference a more competitive alternative to cloud services.

*Source: rss/Simon Willison*

**Background**: llama.cpp is an open-source C/C++ library that enables efficient inference of large language models (LLMs) on consumer hardware, famously starting as a hack to run Meta's LLaMA model using 4-bit quantization on a MacBook. 4-bit quantization is a technique that reduces the numerical precision of a model's parameters from the standard 32 bits to just 4 bits, dramatically shrinking the model's memory footprint and enabling it to run on devices without powerful, specialized GPUs. Hugging Face's Transformers library is the de facto standard framework for publishing, sharing, and using state-of-the-art machine learning models.

### [How I use Claude Code: Separation of planning and execution](https://boristane.com/blog/how-i-use-claude-code/) ⭐️ 7.0/10

A developer published a practical guide detailing a specific workflow for using Anthropic's Claude Code AI tool, advocating for a strict separation between a detailed planning phase and a distinct execution phase to improve reliability. This approach addresses a core challenge in AI-assisted development by structuring interactions to reduce errors and hallucinations, potentially making AI coding agents more predictable and trustworthy for complex, real-world tasks. The guide emphasizes using explicit, forceful language in prompts (like "deeply" and "in great detail") to signal the AI must perform a thorough analysis, and it highlights the need for a clear validation loop and oversight to prevent the agent from going off track.

*Source: hackernews/vinhnx*

**Background**: Claude Code is an agentic AI coding tool from Anthropic that operates in a terminal or IDE, allowing developers to use natural language to edit code, run commands, and manage workflows. The 'plan-and-execute' pattern is a foundational agentic architecture where a comprehensive planning phase is distinctly separated from the action execution phase, which is different from a tight 'Reason and Act' (ReAct) loop.

**Discussion**: Commenters validated the core approach as simple and effective for retaining control, while also expanding on the concept by discussing the need for fine-grained permission boundaries between planning and execution steps. ### [Quoting Thariq Shihipar](https://simonwillison.net/2026/Feb/20/thariq-shihipar/#atom-everything) ⭐️ 7.0/10

Anthropic engineer Thariq Shihipar revealed that prompt caching is a foundational technique enabling the economic viability of Claude Code, a long-running AI agent product, by reusing computation from previous interactions to significantly reduce latency and cost. The company monitors prompt cache hit rates so closely that low rates trigger SEV (Severity) alerts, as they directly impact the ability to offer generous rate limits in subscription plans. This insight matters because it reveals that core business decisions for commercial AI agents, like pricing and rate limits, are directly tied to fundamental engineering optimizations like caching. It highlights that scaling and making AI agent products economically sustainable is not just about model capabilities but also about efficient system architecture that reduces per-query computational costs. The entire Claude Code system harness is built around prompt caching, indicating it's a first-class architectural principle, not just an add-on optimization. A key technical detail is that a high cache hit rate is so critical to operations that it is monitored with production alerts, where a drop is treated as a serious incident (SEV), similar to a performance degradation or outage.

*Source: rss/Simon Willison*

**Background**: Prompt caching is an optimization technique for Large Language Model (LLM) systems where the results of frequently repeated prompt prefixes are stored and reused, avoiding redundant computation. This is crucial because running LLM inferences is computationally expensive and contributes significantly to the cost and latency of AI applications. In production software systems, SEV (Severity) alerts are used to classify and respond to incidents, with lower SEV numbers (like SEV-1 or SEV-2) indicating more critical problems requiring immediate attention. Claude Code is an AI agent product from Anthropic designed to assist with programming tasks.

**Tags**: `#ai-agents`, `#prompt-engineering`, `#llm-optimization`, `#production-systems`, `#anthropic`
